# 概念：余弦相似度

## 💡 先说人话

**余弦相似度就是"两支箭指向同一个方向的程度"。**

两段文字的向量像两支从原点射出的箭。不看箭有多长（文章长短），只看箭头指向哪里（语义方向）。方向越接近，余弦相似度越接近 1，说明这两段话意思越相近。方向差 90 度，相似度就是 0，完全不相关。

用这个简单公式，RAG 就能在几百条纸条里一瞬间找出最相关的几张。

## 📖 一句话定义

余弦相似度（Cosine Similarity）衡量两个向量在方向上的一致性，值域 `[-1, 1]`。值越接近 1 表示方向越一致（语义越相近），越接近 0 表示越不相关。它是 RAG 检索环节的核心度量——"找最相关的内容"本质就是"找余弦相似度最高的向量"。

## ⚙️ 深入原理

### 直觉：比较的是"方向"，不是"位置"

想象从原点出发的两支箭。余弦相似度不看箭有多长（模长），只看它们指向同一个方向的程度：

```
     A ↑
       │   ↗ B      A 和 B 方向接近 → 余弦相似度 ≈ 0.9（高）
       │  /
       │ /
       │/
    ───┼────────→
      /│
     / │
    ↙  │
   C   │           C 和 A 方向几乎相反 → 余弦相似度 ≈ -0.9（低）
```

这个特性对文本向量至关重要：两段文字长短不同，向量"模长"会受影响，但**方向代表语义**。比如：

> "猫是宠物" → 短文本，向量模长较小
> "猫是一种非常受欢迎的家庭宠物，很多人都喜欢养猫" → 长文本，向量模长较大

虽然模长差很多，但它们的**语义方向几乎一致**，余弦相似度依然很高。如果用欧氏距离（比较绝对位置），长文本和短文本会被判为"距离很远"——这显然是错的。

这就是为什么 **RAG 检索用余弦相似度而非欧氏距离**。

### 公式拆解

```
            A · B               Σ(Aᵢ × Bᵢ)           点积
cos(θ) = ─────────── = ─────────────────────────── = ────────
           |A| × |B|     √(ΣAᵢ²) × √(ΣBᵢ²)        模长之积
```

分三步理解：

**第一步：点积（分子）**——两个向量逐维相乘再求和。如果两向量方向一致，正值累加变大；方向不一致，正负抵消变小。

**第二步：模长之积（分母）**——把点积除以两向量各自的长度，消除"长短"影响，只保留"方向"信号。

**第三步：落在 [-1, 1]**——除以模长之积相当于归一化，结果必然在此区间。

### 数值含义速查

| 余弦相似度 | 方向关系 | 通俗理解 |
|---|---|---|
| **1.0** | 完全相同 | "这两句话一个意思" |
| **0.7 ~ 0.9** | 高度相似 | "换了个说法，内核一样" |
| **0.4 ~ 0.6** | 有一定关联 | "有关，但不是一回事" |
| **0.0 ~ 0.3** | 基本无关 | "八竿子打不着" |
| **~ 0.0** | 正交 | "完全没关系" |
| **< 0** | 方向相反 | 文本场景极罕见（对立语义） |

> **注意**：上面的阈值是经验值，不同 embedding 模型的数值分布不同。GLM `embedding-3` 的相似度整体偏高，同义句通常 0.8+，无关句 0.2-0.4。

### 代码实现

```typescript
/**
 * 余弦相似度：A·B / (|A| × |B|)
 * 复杂度 O(D)，D 为向量维度（如 embedding-3 的 2048）
 */
export function cosineSimilarity(a: number[], b: number[]): number {
  let dot = 0;       // 点积 Σ(Aᵢ × Bᵢ)
  let normA = 0;     // A 模长的平方 Σ(Aᵢ²)
  let normB = 0;     // B 模长的平方 Σ(Bᵢ²)

  for (let i = 0; i < a.length; i++) {
    dot += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }

  // 防除零：任一向量为零向量时返回 0
  const denom = Math.sqrt(normA) * Math.sqrt(normB);
  if (denom === 0) return 0;

  return dot / denom;
}
```

**关键细节**：
- `denom === 0` 检查：零向量的模长为 0，直接除会得到 `NaN`。返回 0 是合理的兜底——零向量没有方向，与任何向量都不相似。
- 单次循环完成三个累加：点积、A 模长平方、B 模长平方，避免多次遍历。
- 线性复杂度 O(D)：对 2048 维向量，单次计算约几微秒；在向量库中检索 N 条记录时的复杂度为 O(N×D)。

## 🛠️ 在 Agent 架构中的作用

### 1. RAG 检索的核心度量

向量库的 `search(query, topK)` 本质就是：

```
1. 把 query 变成向量 qVec
2. 对库中每条记录的向量 v，计算 cosineSimilarity(qVec, v)
3. 按相似度降序排列，取前 topK 条
```

正是余弦相似度把"找相关内容"变成了"找最近的向量"——把语义问题转化为了几何计算。

### 2. 相似度阈值：决定"知不知道"

在 RAG 的 System Prompt 中，通常有一个决策逻辑：

| 最高相似度 | 策略 |
|---|---|
| **> 0.8** | 资料高度匹配，直接基于资料回答 |
| **0.4 ~ 0.8** | 有一定关联，回答但标注"仅供参考" |
| **< 0.4** | 资料不相关，回答"根据已有资料无法回答" |

这个阈值是**防止幻觉的关键**——如果没有相关信息，宁可说不知道也不编造。

### 3. 聚类与去重

除了检索，余弦相似度还可以用于：
- **文档聚类**：把相似文档归为一组
- **去重**：相似度 > 0.95 的片段可能是重复内容，可以合并
- **质量评估**：检查检索结果的一致性，评估向量库质量
