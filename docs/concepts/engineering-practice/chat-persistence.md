# 概念：对话持久化

## 📖 一句话定义

对话持久化是将 Agent 的运行时状态（`messages` 数组）从内存中序列化到磁盘，使对话历史能在进程退出后存活、下次启动时恢复。

## ⚙️ 深入原理

### 为什么需要持久化？

程序的两种"记忆"：

```
内存记忆（Step 5）          磁盘记忆（Step 6）
──────────────             ──────────────
位置：RAM                   位置：硬盘/SSD
速度：纳秒级                 速度：毫秒级
生命周期：进程内              生命周期：直到被删除
丢失条件：进程退出/崩溃        丢失条件：文件被删除/磁盘损坏
```

对于对话助手，用户期望像使用聊天软件一样——关掉窗口再打开，对话还在。这就是"持久化"要解决的问题。

### 工作流程

```
┌─ 启动 ─────────────────────────────────────────────┐
│  1. 检查 chat_history.json 是否存在                  │
│  2. 存在 → JSON.parse() → 恢复 messages 数组         │
│  3. 不存在 → 初始化 messages = [system_prompt]        │
└─────────────────────────────────────────────────────┘
         ↓
┌─ 每轮对话 ──────────────────────────────────────────┐
│  1. 用户输入 → push 到 messages                      │
│  2. 发送 messages 给 LLM → 获取回复                   │
│  3. LLM 回复 → push 到 messages                      │
│  4. JSON.stringify(messages) → 写入 chat_history.json │
└─────────────────────────────────────────────────────┘
         ↓
┌─ /clear ───────────────────────────────────────────┐
│  1. messages → 重置为 [system_prompt]                │
│  2. 删除 chat_history.json                          │
└─────────────────────────────────────────────────────┘
```

### JSON 序列化的适用边界

| 能存 | 不能存 |
|------|--------|
| 字符串、数字、布尔 | 函数、类实例 |
| 数组、普通对象 | 循环引用 |
| null | undefined、Symbol、BigInt |

Chat Completion API 的 `messages` 数组恰好是纯 JSON 数据结构（`{role, content}`），天然适合 JSON 序列化。

### 工程化注意事项

- **写入策略**：每轮写（简单，少量数据可接受）vs 批量写（高效，但崩溃可能丢失最后几轮）。原型阶段用每轮写，生产环境用批量或流式追加。
- **文件大小**：对话越长文件越大。JSON 全量读写是 O(n) 的——当 `messages` 达到数千条时，启动加载和落盘都会变慢。生产方案可考虑 SQLite、增量追加（JSONL）或定期归档压缩。
- **并发冲突**：单进程 CLI 无并发问题。如果是 Web 服务，多请求同时写同一个文件会冲突，需要锁或数据库事务。
- **备份与迁移**：JSON 文件易于备份（复制即可）和版本管理（可放入 Git，但注意保护隐私）。

## 🛠️ 在 Agent 架构中的作用

- **Stage 1 的终极交付物**：Step 6 的持久化对话助手是 Phase 1 筑基期的收尾作品——融合了 API 调用、参数控制、流式输出、System Prompt、上下文管理、持久化六大能力。
- **为后续阶段打基础**：Harness Loop（Stage 3）中 Agent 的中间推理步骤、工具调用结果都需要持久化，Step 6 的 JSON 文件模式是最简原型。后续可升级为数据库或向量存储。
- **用户控制权**：`/clear` 体现了"用户应能掌控自己的数据"这一设计原则——持久化不是单向的"永远记住"，而是让用户可以选择遗忘。
