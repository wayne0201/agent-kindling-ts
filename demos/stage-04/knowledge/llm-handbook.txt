大语言模型（LLM）基础手册

一、什么是大语言模型

大语言模型（Large Language Model，简称 LLM）是一种基于深度学习的语言模型，通过在海量文本数据上训练，学习语言的统计规律和世界知识。它的核心能力是"接龙"——给定前文，预测下一个最可能出现的 token。LLM 本身是无状态的：每次调用都是独立的，模型不会记住上一次对话。我们感知到的"记忆"，其实是应用程序把历史对话重新拼进 messages 数组再次发送。

二、Token 与上下文窗口

Token 是 LLM 处理文本的最小单位，大致上 1 个英文单词约等于 1-2 个 token，1 个中文字约等于 1-2 个 token。上下文窗口（Context Window）是模型一次能处理的最大 token 数，例如 8K、32K、128K。上下文窗口限制了模型能"看到"的历史信息量——超过窗口的内容会被截断。这就是为什么长对话需要滑动窗口截断，也是 RAG（检索增强生成）要解决的核心问题：把外部知识按需召回，而不是全部塞进上下文。

三、Prompt 工程

Prompt（提示词）是用户给模型的输入指令。System Prompt 用于设定模型的人格、行为边界和输出格式。好的 Prompt 应该清晰、具体、有结构。结构化输出（如要求模型输出 JSON 或 XML 标签）能让结果可编程解析。温度（temperature）参数控制输出的随机性：低温（0.1-0.3）适合事实问答和代码生成，高温（0.7-0.9）适合创意写作和头脑风暴。

四、Function Calling 与工具调用

Function Calling 让 LLM 从"只会说话"进化到"能动手"。流程分四步：客户端在请求中注册可用工具，模型根据用户意图决定是否调用并返回 tool_calls，客户端执行对应函数，再将结果以 role 为 tool 的消息回传模型。模型本身不执行任何代码——这是刻意为之的安全设计，执行权完全在客户端手中。

五、Harness Loop 与 ReAct

Harness Loop（挽具循环）是 Agent 的执行内核：状态到 LLM 决策到解析执行到观察回写到更新状态，循环往复直到满足终止条件。三种终止条件是 done（任务完成）、max_steps（防死循环）和 error（异常）。ReAct 模式（Reasoning + Acting）要求模型在行动前先输出 Thought 思考，让决策过程可见且更精准。Agent 编程与传统编程的本质区别在于：传统程序的流程是写死的，Agent 的流程是 LLM 每步实时推理出来的。

六、Embedding 与向量检索

Embedding（嵌入）是把文本映射为高维向量的技术，语义相近的文本在向量空间中距离也近。智谱 GLM 的 embedding-3 模型输出 2048 维向量。余弦相似度用于衡量两个向量的方向一致性，值域为 -1 到 1，越接近 1 表示语义越相近。向量检索的流程是：把查询文本向量化，与知识库中所有向量计算相似度，返回得分最高的 topK 条结果。

七、RAG 检索增强生成

RAG（Retrieval-Augmented Generation）= 检索 + 生成。它解决的核心问题是：LLM 的知识有截止日期，且无法访问私域数据。RAG 的流程是：先把文档分块并向量化存入向量库，用户提问时检索最相关的片段，把片段拼进 Prompt 的上下文，让 LLM 基于检索到的内容生成回答。RAG 让模型从"凭记忆回答"升级为"查资料回答"，大幅减少幻觉，且知识可随时更新——只需重新索引文档，无需重新训练模型。

八、文本分块策略

文本分块（Chunking）是 RAG 的关键预处理步骤。分块粒度影响检索精度：整篇文档一个向量太粗，按句分块又太细，段落级（300-500 字符）通常是甜点区。固定长度分块配合重叠窗口（overlap）能防止语义在边界被硬切断。更高级的策略包括按语义边界分块（在段落或句子处切分）和递归分块（先按段落，段落过长再按句子）。
